관리자 대시보드 — 전 은행 이상 거래 통합 모니터링
● 우선순위 상
  • 전체 은행 이상 거래 목록을 한 화면에서 통합 모니터링, 은행 드롭다운으로 전환
  • fraudScore + CASE WHEN SQL로 patternText(거래 패턴 설명) 자동 생성하여 표시
  • 30초 폴링(POLL_INTERVAL_MS = 30,000)으로 신규 이상 거래 자동 갱신
  • 시간대별·월별 이상 거래 시각화 차트 포함, 고위험/중위험/저위험 시나리오별 분류
  • 관리자 전용 접근 제한 — ProtectedRoute로 role 검증, 비관리자 접근 시 리다이렉트
기술 포인트
  • CASE WHEN SQL로 거래 패턴을 텍스트로 자동 생성 — 비주요국 거래 / 심야 이체 / ATM 고액 출금 등 시나리오별 분기
  • setInterval 기반 30초 폴링으로 컴포넌트 언마운트 시 clearInterval 처리하여 메모리 누수 방지
  • 은행 드롭다운 선택 시 동일 API(/api/dashboard/{bankCode}) 재호출로 데이터 전환

TX-001 ~ TX-005
거래 데이터 입력 — 12개 필드 폼 및 AI 예측 연동
● 우선순위 상
  • 12개 필드 입력 폼(userId, bankCode, txChannel, txType, txAmount, txDatetime, txCountry, txLocation, deviceType, deviceOs, deviceId, ipAddress)
  • Controller에서 필드별 null / 빈값 검증 후 Oracle에 저장, fraudScore = 0으로 초기화
  • Oracle 시퀀스(FFG_TX_DATA_SEQ.NEXTVAL)로 TX_ID 자동 채번
  • 저장 완료 후 Flask /predict 호출하여 AI 예측 결과 DB 업데이트
기술 포인트
  • Controller 1차 검증(null/빈값) + Service 2차 검증(비즈니스 규칙)으로 이중 방어 처리
  • TO_DATE(#{txDatetime}, 'YYYY-MM-DD"T"HH24:MI') — ISO 8601 형식 날짜 Oracle 파싱
  • fraudScore 초기 0 저장 후 Flask 예측 결과로 UPDATE 처리하여 응답 지연 분리
